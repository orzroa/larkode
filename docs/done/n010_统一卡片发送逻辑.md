# Task #010: 统一卡片发送逻辑

## 背景

当前代码中卡片生成逻辑分散在多处：
- `src/feishu/__init__.py` 的 `CardBuilder` 类中有多个卡片创建方法
- `src/hook_handler.py` 中有独立的 `create_notification_card` 函数

每处都各自处理：
- 标题映射
- 颜色模板
- 消息编号
- 内容超长处理
- 文件上传判断

导致逻辑重复、维护困难。

## 目标

收敛所有卡片生成逻辑，集中管理卡片类型映射和通用处理逻辑。

## 最终实现：继承模式

### 架构设计

使用 OOP 继承模式：
- `BaseCard` 基类：实现通用的 `build()` 方法，处理所有卡片的共同逻辑
- 子类：只需实现 `build_content()` 方法，构建各自的内容

### 基类实现

```python
class BaseCard(ABC):
    """卡片基类"""

    def __init__(self, card_type: str, return_file_info: bool = False):
        self.card_type = card_type
        self.config = CARD_CONFIG[card_type]
        self.return_file_info = return_file_info

    @abstractmethod
    def build_content(self) -> str:
        """子类实现：构建卡片内容"""
        pass

    def build(self) -> str | Tuple[str, bool, str]:
        """构建卡片 JSON - 所有卡片通用逻辑"""

        # 2. 处理内容超长（通知类型）
        # 3. 构建 NormalizedCard
        # 4. 转换为飞书卡片 JSON
        # 5. 处理文件上传逻辑
```

### 子类实现

```python
class CommandCard(BaseCard):
    def __init__(self, command: str, task_id: str):
        super().__init__("command", False)

    def build_content(self) -> str:
        return f"任务ID: {self.task_id}\n\n命令: `{self.command}`\n状态: 开始处理..."

class TmuxCard(BaseCard):
    def __init__(self, output: str):
        super().__init__("tmux", True)
```

## 统一处理逻辑

在 `BaseCard.build()` 中统一处理：
1. **内容超长截取**：检查 `len(content) > CARD_MAX_LENGTH`
2. **文件上传判断**：`USE_FILE_FOR_LONG_CONTENT` 环境变量
3. **NormalizedCard 构建**：统一的卡片结构
4. **飞书格式转换**：调用 `FeishuPlatform._convert_normalized_card_to_feishu()`
