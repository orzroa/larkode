## 环节拆分说明

### 第1环节：用户发送指令 (可独立测试)
**职责**: 接收用户的初始输入

**输入**: 飞书原始事件

**输出**: 用户原始消息

**测试点**:
- [ ] 各种消息类型的接收
- [ ] 用户ID提取
- [ ] 消息内容解析

---

### 第2环节：响应处理层 (可独立测试)
**职责**: 解析用户命令并决定处理路径

**输入**: 用户原始消息

**处理**:
1. 解析消息内容
2. 判断命令类型
3. 关联上下文
4. 决定处理路径

**输出**: `UserCommandInfo`

**测试点**:
- [ ] 斜杠命令识别
- [ ] 选择回复识别 (1, 2, 3...)
- [ ] escape 识别
- [ ] 普通命令识别
- [ ] 上下文关联

---

### 第3环节：Hook 信息生成层 (可独立测试)
**职责**: 执行 Claude，接收 Hook 事件，生成结构化信息

**输入**: `UserCommandInfo`

**处理**:
1. 创建任务并执行 Claude 命令
2. Claude 处理命令
3. Claude 触发 Hook 事件（如 UserPromptSubmit）
4. 解析 Hook 事件内容
5. 提取关键信息（是否需要选择、选项列表等）
6. 构建结构化事件信息

**输出**: `HookEventInfo`

**测试点**:
- [ ] Hook 事件解析
- [ ] 信息提取完整性
- [ ] needs_selection 判断
- [ ] selection_options 格式验证

**关键理解**:
- 所有命令都经过这个环节
- /select 命令执行后，Claude 触发 Hook 事件，告知需要显示选择菜单
- 这不是特例，而是通过 Hook 机制统一处理

---

### 第4环节：动作决策层 (可独立测试)
**职责**: 根据 Hook 信息决定下一步动作

**输入**: `HookEventInfo`

**处理**:
1. 分析 needs_selection
2. 检查 selection_options
3. 分析 event_type
4. 选择动作类型

**输出**: `ActionDecision`

**测试点**:
- [ ] 不同 Hook 信息的动作选择
- [ ] 边界条件处理
- [ ] 默认动作逻辑

---

### 第5环节：响应构建层 (可独立测试)
**职责**: 根据动作类型构建飞书卡片

**输入**: `ActionDecision`

**处理**:
1. 选择卡片类型
2. 构建卡片 JSON
3. 填充内容
4. 验证格式

**输出**: `CardJSON`

**测试点**:
- [ ] 各种卡片类型的构建
- [ ] JSON 格式验证
- [ ] 内容正确性

---

### 第6环节：发送执行层 (可独立测试)
**职责**: 将卡片发送给飞书用户

**输入**: `CardJSON`

**处理**:
1. 验证卡片格式
2. 获取访问令牌
3. 发送 API 请求
4. 处理响应

**输出**: `SendResult`

**测试点**:
- [ ] API 调用正确性
- [ ] 错误处理
- [ ] 结果记录

---

## 完整流程示例

### 示例：Claude 需要选择

```
[环节1] 用户发送 "帮我读取 /etc/passwd"
   ↓
[环节2] 响应处理:
   ├─ 识别为普通命令
   └─ needs_claude = true
   ↓
[环节3] Hook 信息生成:
   ├─ 执行 Claude 命令
   ├─ Claude 识别需要权限
   ├─ Claude 触发 UserPromptSubmit Hook
   │  ├─ event_type = "user_prompt_submit"
   │  ├─ needs_selection = true
   │  ├─ selection_options = [权限选项...]
   │  └─ selection_type = "single"
   └─ 生成 HookEventInfo
   ↓
[环节4] 动作决策:
   └─ action_type = "SEND_SELECTION_MENU"
   ↓
[环节5] 响应构建:
   └─ 创建权限选择菜单卡片
   ↓
[环节6] 发送执行:
   └─ 发送卡片给用户
   ↓
[飞书用户] 显示权限选择菜单
   ↓
[用户回复] "1" (选择 "申请权限")
   ↓
[回环] 回到环节1
   (继续执行 Claude 命令，根据选择执行相应操作)
```

## 总结

整个 Hook 触发流程包含 **6个环节**：

| 环节 | 名称 | 职责 | 可独立测试 |
|--------|------|------|-------------|
| 1 | 用户发送指令 | 接收用户初始输入 | ✅ |
| 2 | 响应处理层 | 解析命令，决定处理路径 | ✅ |
| 3 | Hook信息生成 | 执行Claude，接收Hook事件，生成结构化信息 | ✅ |
| 4 | 动作决策层 | 根据Hook信息决定动作 | ✅ |
| 5 | 响应构建层 | 构建飞书卡片 | ✅ |
| 6 | 发送执行层 | 发送卡片给用户 | ✅ |

**关键特性**:
- 所有命令都走完整流程，**没有特例**
- /select 命令也是通过 Hook 机制触发选择菜单
- 用户回复后直接**回环到环节1**，形成闭环
- 每个环节都可以**独立测试**

**核心理解**:
- Claude 通过 Hook 事件告知系统需要什么（如需要用户选择）
- 系统根据 Hook 信息构建并发送相应的卡片
- 用户选择后回到第1环节，形成完整的交互循环
